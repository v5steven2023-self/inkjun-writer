# Rewrite Few-shots

## 适用范围

仅用于起草模式和改稿模式。

## 分级定义

- **L0**：必须修改，输出前全部清零。典型问题：先否定再肯定的转折骨架、主语缺失、破折号补充说明。
- **L1**：建议修改，每段不超过 2 处。典型问题：总结式心理词汇、万能过渡句。
- **L2**：可选优化，不影响 Gate 通过。典型问题：冗余限定词、句子节奏过于均匀。

## 门禁清单（输出前必须满足）

- **Gate 1**：全文 L0 条目数必须为 0
- **Gate 2**：每一段 L1 条目数 <= 2
- **Gate 3**：核心观点、事实与数据未被改动（最小改动原则）

## Few-shot

| L0 | 问题不在错别字，也不在观点跑偏，而在整篇文章虽然都说对了，却不像我写的。 | 这篇文章没有明显错误，却还是不像我写的。 | 避免先否定片面观点，再肯定正面观点。而是直接表达观点。 |
| L0 | 这不是文笔问题，而是流程问题。 | 真正卡住我的，是写作流程。 | 避免先否定片面观点，再肯定正面观点。而是直接表达观点。 |
| L0 | 信息没错，语气也算顺，但发出去后很快就收到一句反馈：像 AI 写的。 | 我原本觉得这篇文章写得还行，结果发布后不久，就有朋友回复我「你这篇文章像 AI 写的」。 | 句子主语缺失，读者无法判断动作主体。 |

| L1 | 让我警觉的是另一次。我让它把一篇长文改简洁，它把全文压缩得只剩结论，干得像说明书。 | 还有一次更离谱。我让它把一篇长文改简洁，它直接把全文压缩得只剩结论，干得像说明书。 | 避免使用「警觉/心里一震」等总结式心理词汇。 |
| L1 | 答案藏在「AI 不是人」这件事里。 | 说到底，还是因为 AI 不是人。 | 避免使用「答案藏在/本质上/更重要的是」等万能过渡句。万能过渡句看起来在推进文章，其实没有提供新信息。 |
| L2 | 这是第一个阶段。 | 直接删除。 | 通过小标题，读者会自动判定到哪个阶段了，冗余解释。 |
| L2 | 从某种程度上来说，这个判断是对的。 | 这个判断是对的。 | 去掉模糊限定词。L2 不影响 Gate 通过，可选修改。 |
